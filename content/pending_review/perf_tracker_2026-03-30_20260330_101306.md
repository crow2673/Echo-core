# Building on Success: Local AI and Automation with Ollama and Notion

After my last project where I gave my local AI a public brain using Echo and Notion, I wanted to dive deeper into how I could make it more dynamic and interactive. Specifically, I wanted to focus on automating responses to trending topics like "she," "started," and "fixing." ## Initial Setup

I started with my usual setup: a Ryzen 9 5900X, RTX 3060 12GB, 32GB RAM, and Ubuntu Linux. I run Qwen2.5:32b via Ollama locally, without Docker or cloud services, to avoid API costs.

First, I needed to ensure Ollama was properly installed and running. If you haven't set it up yet, you can do so with:

```bash

```

Once Ollama was up and running, I focused on integrating it with my existing Echo setup.

## Tackling Trending Topics

My goal was to have my local AI automatically respond to trending topics like "she," "started," and "fixing." I started by setting up a webhook in Echo to listen for these keywords. However, I quickly ran into issues with the webhook setup. The initial configuration was straightforward:

```bash
echoctl webhook create --name "trending-topics" --url "http://localhost:8080/webhook"
```

But the webhook didn't trigger as expected. After some debugging, I realized the webhook URL needed to be updated to reflect the actual endpoint I was using. This took longer than expected, but it was a good reminder to always double-check configuration details.

## Integrating Ollama with Notion

Next, I focused on integrating Ollama with Notion to create a more dynamic and interactive system. I wanted to have the AI generate responses and store them in Notion for easy access.

I started by setting up a Notion database to store the trending topics and their responses. Here’s how I created the database using the Notion API:

```python
import requests

# Notion API token
NOTION_TOKEN = "your_notion_token"
DATABASE_URL = "https://api.notion.com/v1/databases"

# Create the database
headers = {
    "Authorization": f"Bearer {NOTION_TOKEN}",
    "Content-Type": "application/json",
    "Notion-Version": "2022-06-28",
}

data = {
    "parent": {"database_id": "your_database_id"},
    "title": [{"type": "text", "text": {"content": "Trending Topics"}}],
    "properties": {
        "Topic": {"title": {}},
        "Response": {"rich_text": {}},
    },
}

response = requests.post(DATABASE_URL, headers=headers, json=data)
print(response.json())
```

This setup worked, but I faced some issues with the API rate limits. I had to adjust the script to handle retries and delays to ensure smooth operation.

## Automating Responses

With the database set up, I needed to automate the generation of responses. I used Ollama to generate responses based on the trending topics:

```python
import requests

def generate_response(topic):
    response = requests.post("http://localhost:8080/api/generate", json={"prompt": topic})
    return response.json()["response"]

topic = "she started fixing"
response = generate_response(topic)
print(response)
```

This was where things got tricky. The initial responses were either too generic or didn't align well with the trending topics. I had to tweak the prompts and fine-tune the model to get better results. After several iterations, I managed to get the AI to generate more relevant and engaging responses.

## Performance and Usage

Since the initial setup, I've been running several AI models and tools on this machine, including Qwen 2.5:32b via Ollama. Here are some practical examples and insights:

### Running Qwen 2.5:32b

To get Qwen 2.5:32b running locally, I followed the instructions provided by Ollama:
```sh
curl -fsSL https://ollama.com/install.sh | sh
```

This command installed the necessary dependencies and set up the environment. Once installed, I could start the model with:
```sh
qwen -c 2 -t 1000 -m 4096 -b 8 -l 4096 -s 1024 -t 1 -p 0.95
```

This command runs the model with specific parameters that I found to be effective for my use cases. The `qwen` command itself is straightforward and provides a lot of flexibility.

### Performance Metrics

Here are some performance metrics I've noted over time:

- **CPU Usage**: Running Qwen 2.5:32b on the Ryzen 9 5900X shows consistent high CPU utilization, especially when the model is processing large inputs.
- **GPU Usage**: The RTX 3060 handles the computational workload well, but I've noticed that the GPU usage spikes when the model is running intensive tasks.
- **RAM Usage**: With 32GB of RAM, I haven't experienced any out-of-memory issues, even when running multiple models or tools simultaneously.

### Adjustments and Optimizations

Since the initial setup, I've made a few adjustments to optimize performance and resource management:

- **Resource Monitoring**: I use tools like `htop` and `nvidia-smi` to monitor CPU and GPU usage. This helps in identifying bottlenecks and optimizing the model parameters.
- **Model Parameters**: I've experimented with different parameters to find the best balance between performance and resource usage. For instance, reducing the batch size (`-b`) and increasing the sequence length (`-l`) can sometimes improve performance.
- **Cooling Solutions**: Given the heat generated by the CPU and GPU, I've added a small air cooler to ensure the system runs cool and stable.

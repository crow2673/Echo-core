## Tackling 'Pending' and 'Review' Tasks

My initial goal was to have my AI automatically categorize tasks as 'pending' or 'review' based on certain criteria. I started by creating a Python script that would parse through my task list in Notion and update the status based on the AI's analysis.

### Step 1: Setting Up the Environment

First, I needed to ensure that my environment was set up correctly. I installed the necessary Python packages for interacting with Notion and for running the AI:

```bash
pip install requests
pip install ollama
```

### Step 2: Parsing and Updating Tasks

I wrote a script to parse through tasks and update their status:

```python
import requests
import ollama

def get_tasks():
    # Notion API call to get tasks
    response = requests.get('https://api.notion.com/v1/tasks', headers={'Authorization': 'Bearer YOUR_NOTION_TOKEN'})
    return response.json()

def update_task(task_id, status):
    # Notion API call to update task status
    requests.patch(f'https://api.notion.com/v1/tasks/{task_id}', json={'status': status}, headers={'Authorization': 'Bearer YOUR_NOTION_TOKEN'})

def analyze_task(task):
    # Using Ollama to analyze the task
    analysis = ollama.analyze(task)
    if analysis['priority'] > 70:
        return 'review'
    else:
        return 'pending'

tasks = get_tasks()
for task in tasks:
    new_status = analyze_task(task)
    update_task(task['id'], new_status)
```

### Struggles and Failures

Initially, the script worked, but it was too slow and didn't handle edge cases well. I had to optimize the API calls and handle exceptions more gracefully. The biggest challenge was integrating the Ollama analysis with the Notion API calls. It took several iterations and debugging sessions to get the logic right.
